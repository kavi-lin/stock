## Root Cause Hypotheses

### Hypothesis A — Bull regime 下 protocol 把「合理估值」當「不買」(對應 Pattern 1, 2)

- **推論**：mega-cap 連續 +9% 到 +90% 期間 system 給 HOLD 或 score 0.5–1.0 → 顯示 valuation lane 把 forward multiple 拉高的 stock 視為「貴，HOLD」，但 momentum / sector tail-wind 在 bull regime 下會繼續推升。BUY 門檻在現行 weighted blend 下需要 score ≥1.5，等於要求三個 lane 同時強信號才動手；mega-cap 在熱潮中通常 fundamentals 領先而 valuation 拖後腿，永遠卡 1.0–1.5
- **證據引用**：`deep-dive_AMZN_2026-04-11` BUY score 1.267 → hit (+13.7%) vs `deep-dive_NVDA_2026-04-22` HOLD score 0.386 → miss (+6.3%)；`deep-dive_QCOM_2026-04-26` HOLD score 0.732 → miss (+45.8%)

### Hypothesis B — Indexer / report parser 與 V5.0 schema 不同步 (對應 Pattern 3)

### Hypothesis C — momentum-screen 因子組合在當前 breadth-narrow 下無 alpha (對應 Pattern 5)

## 系統盲點

- **Verdict 規則本身**：deep-dive 的 verdict 「HOLD → 上漲 = miss」這個定義過嚴。若是 risk-managed avoidance（避開 -10% drawdown 但漲了 +5%），仍判 miss，會讓系統誤以為 HOLD 全錯。建議區分 "miss-with-low-cost" vs "miss-with-high-opportunity-cost"
- **無 macro_regime 欄位**：所有 deep-dive `tuning_hooks.macro_regime` 為 null（grep `"macro_regime": null` 全中），無法做 regime-conditioned 統計，Hypothesis A2 / Rec 2 都因此停在 "假設" 階段
- **earnings-analyzer / short-term-weekly / postmortem 各 n=1**：完全無法評估
- **window_complete_pct < 50% 共 49 筆**：43% 的 deep-dive verdict 是在 window 未滿時 freeze 的，後續可能反轉，需要 deferred re-eval 機制
- **無 verdict 對應 P&L magnitude**：missed +90% (INTC) 與 missed +5% (TSM) 在 verdict 上等價計 1 miss，掩蓋了真正影響 portfolio 的決策
