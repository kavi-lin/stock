### Pattern A — deep-dive HOLD/CANCEL 系統性錯過上漲 (n=13, robust)

- 證據：13 個 HOLD/CANCEL → miss 全部都是「stock went up，system 沒進」。例：`deep-dive_INTC_2026-04-18` HOLD → +51.6%；`deep-dive_AMD_2026-04-18` HOLD score 0.413 → +31.1%；`deep-dive_TSLA_2026-04-07` HOLD → +12.7%（83% window）；`deep-dive_CRWV_2026-04-10` CANCEL → +16.7%（73% window）。
- 統計：6/13 已達 ≥50% window（mature），不全是早期假訊號。CANCEL/CANCEL(HOLD) median return = **+6.75%**（n=8），HOLD median = +1.04% (n=17)。BUY median = +3.06% (n=15)。
- 反直覺對比：'miss' verdict 平均 +11.55% / median +7.6%；'hit' verdict 平均 -0.98% / median -0.52%。系統的「正確」是當 stock 沒漲時觀望對；錯誤都集中在「stock 漲了卻沒進」。

- 統計：22 個 bullish action vs 26 個 HOLD/CANCEL。BUY 既沒系統性領先 HOLD（差 ~2%），也沒避開大跌。

## Root Cause Hypotheses

### H1 → Pattern A：保守偏誤 + Cycle/Macro 多空模板未跟上 RISK_ON

- 對應 pattern：A、B、D
- 推論：4 月整體呈現恢復性上漲（all-deep-dive median return +1.35%、AMD/INTC/TSM/MRVL 群族雙位數）。系統 final_action 的 HOLD/CANCEL 比例（26/48 有 action 的 record，54%）超過 BUY/EXECUTE（22/48，46%），但實際分布顯示 BUY 沒比 HOLD 領先。可能來自：(a) Bears agent 在 News-driven 風險評估時權重過高；(b) Trader proposal 的 entry 條件太嚴（要 -5% 拉回才觸發）。
- 需要：cross-check `investment/protocol/weights_table.yaml`（如有）跟 trader_proposal 的 entry 邏輯。

### H2 → Pattern B：BUY/HOLD/CANCEL 三類 forward return 高度重疊 = 投票機制鑑別力不足

- 對應 pattern：B
- 推論：final_score 與 forward return 沒明確分離。可能是 4 個 agent 信號相關性高（同期看 Fundamentals、News、Technical、Sentiment 都被同一波 macro 訊號 dominate），最終 score 變平。也可能是 score → action 的閾值設定（>0.5 BUY、<-0.5 SELL）過於對稱，無視當前 macro_regime。
- 需要：跑 cross-correlation：4 agent score 的 pairwise correlation；如 ρ>0.7 表示 agent 多餘。

### H3 → Pattern C：momentum-screen 沒過濾 overbought 就讓進 Top-N

- 對應 pattern：C
- 推論：score_top 80+ 配 overbought_rsi=10/20 + parabolic_blowoff_risk=5-7 進 Top-N，其後 5d 平均回吐。可能是 score 公式對 RSI 處理不夠重，或 Top-N 篩選沒有用 warning_count 做硬性扣分。
- 需要：對 overbought_rsi/parabolic 旗標 stocks 做 forward 5d return vs 沒 warning 的對比。

### H4 → Pattern D：sector-scan rating_skew 規則跟 regime 是兩條獨立邏輯

- 對應 pattern：D
- 推論：「RISK_ON 但 rating_skew=defensive」表示 regime 判斷依據（macro/breadth）跟 sector rating 計算（個別 sector relative momentum）使用不同 inputs，輸出彼此不對齊。可能 sector rating 用週級 momentum，regime 用日級 macro，時序錯位。
- 需要：抓最近 2 週 sector rating 計算 vs regime label 的時間對齊報表。

### H5 → Pattern F：news-digest 沒有強制要求 schema 必填 macro_delta

- 對應 pattern：F
- 推論：digest_output_schema 可能把 macro_delta 標 optional，導致 60% 樣本無 verifiable claim。
- 需要：檢查 `news/digest_output_schema.md`，validator gate 沒卡 macro_delta 必填。
