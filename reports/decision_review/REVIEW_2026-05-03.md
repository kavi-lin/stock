## Root Cause Hypotheses

### H1 — score→action 映射不完整或被高門檻擋住 (對應 Pattern 2, 5)
- 47/102 deep-dive `final_action=None`，且其中很多是 score 0.5-1.5 區間的 weak buy
- 假設：protocol Phase 5 的 action emit 規則需要 `score≥X` 且 `agent confidence≥Y` 才出 BUY，當前 X/Y 太嚴
- 證據：MU 連續 3 次 score 1.1-1.2 不出 action，但 BUY 訊號歷史 hitR 52%
- 推論：放寬 BUY 門檻 (例如 score≥0.8 即 emit BUY) 可直接收回最大宗 miss
- 需要的額外資料：Phase 5 emit 規則的 config 來源、score 門檻參數

### H2 — Fundamentals agent 在 mega-cap 動能股上有結構性偏空 (對應 Pattern 3)
- TSLA/MSFT/AAPL/AVGO/MU/SMR 的 Fundamentals SELL/HOLD 全錯
- 假設 (a)：Fundamentals 用 valuation 為主要 input，當前市況「貴的更貴」邏輯主導
- 假設 (b)：Fundamentals 信心過高 (TSLA conf 0.78) 在 voting 中過度壓制其他 BUY 信號
- 推論：在 macro_regime=BROADENING/CONCENTRATION 下 Fundamentals 權重應降
- 需要的額外資料：weights table 是否依 regime 切換、Fundamentals score 計算公式

### H3 — Technical agent 短期回檔訊號被當長期信號使用 (對應 Pattern 4) [N=4 初步]
- LITE/AMD/AAOI 都在動能股上給 SELL，4 月後續強勢上漲
- 假設：Technical 看 daily/weekly 過於 reactive，回檔即觸發 SELL，未對齊 30d 評估窗
- 推論：Technical 訊號需匹配 verdict 評估期 (30d)，加 trend filter
- 需要的額外資料：Technical agent 用的時間框

### H4 — momentum-screen verdict 標準過嚴或評估窗錯配 (對應 Pattern 6)
- 11 筆 0 hit，但 hit 標準是 ≥10/20 對方向？

- 需要的額外資料：`scripts/build_event_index.py` 中 momentum-screen verdict 計算函數

### H5 — Indexer parsing regression (對應 Pattern 10) [system bug]
